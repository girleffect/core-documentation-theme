Components

Remember that can change these attributes to whatever you like. You can read more here about implementing your own theme.

Global

We use the following global elements:

  • * for box-sizing
  • html, body for resets, global font sizes and colours
  • .Container is the main wrapper which contains .Header, .Content and .Footer

Header

The header has 3 main elements: branding, title and links, with one or two children:

  • .Header is the parent container where you can define things like background colour and images.
  • .Header-branding is the container of the logo.
  • .Header-branding-image is the class of the logo.
  • .Header-title is the container of the h1 in the title.
  • .Header-links is an extra section for navigational links.

Footer

Footers are generally pretty straight forward, so it only has the .Footer parent class with one p child element which contains the text.

Typography

We use the following typographic elements throughout:

  • body, textarea, select for resets & globals, add or remove at will
  • a, a:visited for links
  • h1 for page titles
  • h2 for sub titles
  • p, ul for text
  • hr for dividers
  • .center for centre-aligning text

You might want to have more than that, especially for enhanced phones, so you can add anything you want. For instance:

  • @font-face for custom fonts which you'll just reference in your elements

List

The .List element is used in the OIDC Authorize page. It's just a text ul, but the classes are there for you to change:

  • .List is the parent class
  • li is the child, which contains strong and small elements

Tokens

The .Tokens element is used in the 2FA Tokens page. It's just a text ul, but the classes are there for you to change:

  • .Tokens is the parent class
  • li is the child

Forms

There are quite a few forms, and most form elements that you are used to get used one way or another:

  • .Form is the parent class of any form
  • .Form-item is the parent class of every form item, and contains the .Field-label and .Field-item elements
  • Please note the .Field--required class (for validation) is a sibling of .Form-item. Everyone likes to surface errors in different ways, so this provides that added flexibility
  • .Field-item is the parent of the input or select etc
  • Please note upon server-side validation, 2 more child elements get created inside .Field-item. They are .errorlist which is a ul that contains the errors, and .Field-message which might contain some text about the error or an additional validation message.

Messages

There are general messages used for success and errors:

  • .Messages is the parent class. It's a div element.
  • Each message is a div element and has a .Message class.
  • The modifiers for success and error messages are .Message--success and .Message--error.

Buttons

You can probably argue buttons should be part of forms, but they can be used outside forms too:

  • .Form-buttons is the parent class of the button container in a form
  • .Button is the class of an input or any a that appears like a button

Additional

Below forms there are sometimes a need for additional navigation or supplementary text:

  • .Additional-text is applied on p elements below forms, where applicable

Menu

The menu is located at the bottom of some pages (where required, like on the Edit Profile page):

  • .Menu is the parent class of the menu.
  • .Menu-title is a child of .Menu and the menu's title
  • .Menu-list is a child of .Menu and the parent of the menu items. It is a ul element.
  • .Menu-item is a child of .Menu-list and the element of each menu item. It is a li element.

QR

The QR image gets generated within the 2FA section. The elements are:

  • .QR is the parent and is a p element.
  • .QR-image is a child of .QR and is an img element
